1. Introduction

bio.dev is a fullstack, multi-tenant portfolio builder that lets users create professional portfolio pages in minutes — no coding required.

🎨 Glassmorphism UI

Modern frosted-glass design with smooth animations

📊 Built-in Analytics

Page views, referrers, unique visitors — no third-party needed

🌍 Bilingual Support

TR / EN content with language toggling for visitors

🚀 Live Preview

See changes in real-time as you edit in the dashboard

📱 Fully Responsive

Optimized for desktop, tablet, and mobile devices

🔒 Secure Auth

NextAuth.js with JWT sessions and bcrypt hashing

How it works: Each registered user gets a unique portfolio URL at yourdomain.com/username. They manage all their content — profile, projects, skills, contact info — from a single-page dashboard with a split-panel live preview.

3. Installation & Setup

1
Extract the files

Unzip the downloaded package and navigate to the source-code directory.

2
Install dependencies 
npm install
3
Configure environment 
cp .env.example .env.local

Edit .env.local with your database URL and auth secret. See Configuration for details.

4
Set up the database 
# Create database tables
npx prisma db push

# (Optional) Seed with sample data
node scripts/seed.js
# Creates test user: test@bio.dev / test123
# Public page at: http://localhost:3000/test
5
Start the development server 
npm run dev
# → Open http://localhost:3000
Quick Start: For the fastest setup, use SQLite (already configured in .env.example). No external database needed — just run npm install, npx prisma db push, and npm run dev.

4. Configuration

Environment Variables

VariableRequiredDescription
DATABASE_URL✅Database connection string. SQLite: file:./prisma/dev.db · PostgreSQL: postgresql://user:pass@host:5432/db
AUTH_SECRET✅Random secret for encrypting JWT sessions. Generate with: openssl rand -base64 32
AUTH_URLNoApplication URL. Auto-detected on Vercel. Set manually for other hosts.

Database Options

DatabaseUse CaseDATABASE_URL Format
SQLiteLocal development, demosfile:./prisma/dev.db
PostgreSQLProduction, multi-userpostgresql://user:password@host:5432/dbname
Important: When switching between SQLite and PostgreSQL, update the provider field in prisma/schema.prisma accordingly ("sqlite" or "postgresql"), then run npx prisma db push to create the tables.

5. Features Overview

Multi-Tenant Architecture

Each user owns a unique Site identified by a slug. The public portfolio lives at /username, while all management is done at /dashboard. Complete data isolation between users.

Authentication

  • Email/password registration with real-time slug availability checker
  • JWT session strategy via NextAuth.js v5
  • Middleware-protected dashboard routes
  • bcrypt password hashing (12 salt rounds)
  • Reserved slug blocking (admin, dashboard, api, etc.)

Public Portfolio

  • Hero section — Title + animated typing lines
  • Skills — Categorized pill-style skill badges
  • About — Markdown-rendered biography
  • Featured projects — Highlighted work with tech stack pills
  • Case study pages — Problem → Solution → Results format
  • Contact/social links — Customizable CTA buttons
  • Language toggle — TR/EN switcher for visitors
  • Framer Motion animations — Fade-in on scroll, typing effect, hover interactions

Visitor Analytics

  • Cookie-free tracking (fingerprint-based, GDPR-friendly)
  • Total views, unique visitors, daily breakdown
  • Top referrer sources, most visited pages
  • Configurable time period (7/14/30 days)

SEO Optimization

  • Server-side rendered public pages
  • Dynamic metadata (OpenGraph, Twitter cards)
  • Semantic HTML structure
  • Security headers (X-Frame-Options, CSP, HSTS-ready)

6. Dashboard Guide

The dashboard uses a split-panel layout: editor on the left (50%), live preview on the right (50%).

Profile Tab

  • Edit hero title, subtitle, and typing animation lines
  • Manage skill categories (add/remove categories, comma-separated items)
  • Write your about section in Markdown
  • Configure contact/social link buttons
  • Auto-save with 800ms debounce — changes save as you type
  • TR/EN language toggle to edit content in both languages

Projects Tab

  • Create, edit, and delete portfolio projects
  • Case study fields: Problem, Solution (JSON array), Results (JSON array)
  • Toggle Featured and Published status per project
  • Tech stack as JSON array, cover image URL
  • Bilingual editing (TR/EN)

Settings Tab

  • Site title, slug, default language, publish/unpublish
  • Contact info (email, LinkedIn, GitHub, website, CV URL)
  • Professional metrics (years experience, projects delivered, industries)
  • Availability status (Available, Busy, Unavailable)

Analytics Tab

  • Total views, period views, unique visitors (metric cards)
  • Daily views bar chart with period selector
  • Top referrer sources and most visited pages

7. Customization

Theme Colors

The accent color is defined in app/globals.css. Change the CSS custom properties to match your brand:

:root {
  --accent: #4f46e5;            /* Main accent (currently indigo) */
  --accent-light: rgba(79, 70, 229, 0.10);
  --accent-mid: rgba(79, 70, 229, 0.22);
}

Typography

Fonts are loaded in app/layout.tsx via Google Fonts. Default setup:

  • Body text: Inter (weights 300–700)
  • Headings: Space Grotesk (geometric, modern)

To change fonts, update the <link> tags in the layout head and the corresponding CSS classes.

Adding New Languages

  1. Create a new dictionary file: lib/i18n/fr.ts (copy from en.ts)
  2. Add the language to the context in lib/i18n/LanguageContext.tsx
  3. Update the language toggle in the public portfolio component
  4. Add corresponding *Fr columns to the Prisma schema (optional, for user content)

Extending the Schema

To add new fields or models:

# 1. Edit prisma/schema.prisma
# 2. Apply changes
npx prisma db push     # Dev (no migration history)
npx prisma migrate dev  # Prod (creates migration file)

8. Deployment

Vercel (Recommended)

1
Push to GitHub

Initialize a git repository and push your code.

2
Import in Vercel

Go to vercel.com/new and import your repo.

3
Set environment variables

Add DATABASE_URL (PostgreSQL) and AUTH_SECRET in Project Settings → Environment Variables.

4
Deploy

Build command is pre-configured: prisma generate && prisma migrate deploy && next build

Vercel will auto-deploy on every push.

Database: For production, use Vercel Postgres, Neon, or Supabase for a managed PostgreSQL database. Remember to update the provider in schema.prisma to "postgresql".

Self-Hosted / VPS

# Build for production
npm run build

# Start production server
npm start
# Runs on port 3000 by default

9. FAQ & Troubleshooting

How do I switch from SQLite to PostgreSQL?

  1. Update provider in prisma/schema.prisma from "sqlite" to "postgresql"
  2. Set DATABASE_URL in .env.local to your PostgreSQL connection string
  3. Run npx prisma db push to create tables
  4. Run node scripts/seed.js to create test data (optional)

I get "AUTH_SECRET is missing" error

Make sure AUTH_SECRET is set in your .env.local file. Generate one with: openssl rand -base64 32

The dashboard shows a blank page

Check: (1) You are logged in, (2) Your database is running and seeded, (3) The DATABASE_URL is correct.

Can I use MySQL instead of PostgreSQL?

Prisma supports MySQL. Change the provider to "mysql" in the schema and adjust the DATABASE_URL format accordingly. Some field types may need adjustment.

How do I add a custom domain?

On Vercel: Project Settings → Domains → Add your domain. For self-hosted: configure your reverse proxy (Nginx, Caddy) to point to port 3000.
